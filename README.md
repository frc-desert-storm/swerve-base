# GitHub Workflow

| #   | Action                                                                 | Command(s)                                  | Notes                                                                                                                                                          |
| --- | ---------------------------------------------------------------------- | ------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Get the repo on your computer                                          | `git clone <repo-url>`                      | Run this **once per computer**. Copies the GitHub repo to your machine.                                                                                        |
| 2   | Go into the project folder                                             | `cd <repo-folder>`                          | Move into the cloned repo directory before any other Git commands.                                                                                             |
| 3   | Make sure `main` is up to date                                         | `git checkout main`                         | Switch to the `main` branch. Safe if you have no uncommitted changes.                                                                                          |
|     |                                                                        | `git pull`                                  | Download the latest code from GitHub into `main`. **Always do this at the start of a work session.**                                                           |
| 4   | Create your own branch                                                 | `git checkout -b <your-branch-name>`        | Creates a new branch **from main** and switches to it. Example: `git checkout -b nathan/add-swerve-comments`                                                   |
| 5   | Write code / make changes                                              | _(edit files in VS Code / IDE)_             | Add comments, fix bugs, add features. Save your files.                                                                                                         |
| 6   | See what changed                                                       | `git status`                                | Shows changed files and which ones are tracked/staged. Use this often.                                                                                         |
| 7   | Stage your changes                                                     | `git add .`                                 | Stages **all** modified files. Use `git add <file>` for specific files.                                                                                        |
| 8   | Commit your work locally                                               | `git commit -m "Describe what you changed"` | Saves a snapshot in your **local** repo. Not yet on GitHub. Use a clear message like: `"Add comments to SwerveDriveModule"`                                    |
| 9   | Push your branch to GitHub                                             | `git push -u origin <your-branch-name>`     | Creates the branch on GitHub and uploads your commits. The `-u` links your local branch to the remote one.                                                     |
| 10  | (If main changed while you worked) Update your branch with latest main | `git checkout main`                         | Switch back to `main`.                                                                                                                                         |
|     |                                                                        | `git pull`                                  | Get latest changes from GitHub into `main`.                                                                                                                    |
|     |                                                                        | `git checkout <your-branch-name>`           | Go back to your branch.                                                                                                                                        |
|     |                                                                        | `git merge main`                            | Merge any new changes from `main` into your branch. Resolve conflicts if needed, then `git add` + `git commit` again.                                          |
| 11  | Create a Pull Request (PR)                                             | _(On GitHub website)_                       | Go to the repo → **Pull Requests → New Pull Request**, choose your branch as the “compare” branch. Or click **“Compare & pull request”** banner after pushing. |
| 12  | Fill out the PR details                                                | _(On GitHub)_                               | Add a **title** and **description** of what you changed. Assign the **admin/mentor** as reviewer. Click **Create Pull Request**.                               |
| 13  | Admin review & merge                                                   | _(Admin)_                                   | Admin reviews code, requests changes if needed, then **Merge Pull Request** into `main`.                                                                       |
| 14  | Everyone updates their main                                            | `git checkout main`                         | All team members do this after a merge.                                                                                                                        |
|     |                                                                        | `git pull`                                  | Get the merged code from GitHub so everyone is in sync.                                                                                                        |
| 15  | Start next task from fresh main                                        | Go back to **Step 4**                       | For a new feature or fix, always create a **new branch** from the freshly updated `main`.                                                                      |
